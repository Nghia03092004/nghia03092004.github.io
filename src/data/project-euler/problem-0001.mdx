This is a good first entry because the brute-force solution is trivial, but the cleaner solution exposes the pattern that matters later.

## The real move

Instead of summing every multiple directly, split the problem into:

- multiples of 3
- multiples of 5
- subtract multiples of 15 once

That turns the problem into three arithmetic sums.

## Why I still keep the brute-force version

The brute-force version is useful as a correctness oracle.

```python
total = sum(n for n in range(1000) if n % 3 == 0 or n % 5 == 0)
print(total)
```

Once the direct version is correct, I replace the work with the closed-form approach and keep the note about why inclusion-exclusion is the decisive idea.
