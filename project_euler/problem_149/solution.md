# Problem 149: Searching for a Maximum-sum Subsequence

## Problem Statement

Find the greatest sum of any contiguous subsequence in any direction (horizontal, vertical, diagonal, anti-diagonal) in a 2000 x 2000 table generated by a lagged Fibonacci generator.

## Mathematical Development

### Generator

The table entries $s_k$ for $k = 1, \ldots, 4000000$ are defined by:

- For $1 \le k \le 55$: $s_k = (100003 - 200003k + 300007k^3) \bmod 10^6 - 500000$
- For $k > 55$: $s_k = (s_{k-24} + s_{k-55} + 10^6) \bmod 10^6 - 500000$

The table is filled row by row: entry at row $i$, column $j$ (1-indexed) is $s_{1000(i-1)+j}$. Wait -- for a 2000x2000 table: $s_{2000(i-1)+j}$.

Actually: $s_k$ for $k = 1, \ldots, 4000000$ fills the 2000x2000 table. Entry $(i,j)$ = $s_{2000(i-1)+j}$.

### Directional Maximum Search

For each of the four directions, apply **Kadane's algorithm** to find the maximum contiguous subsequence sum:

1. **Horizontal**: Apply Kadane's to each of the 2000 rows.
2. **Vertical**: Apply Kadane's to each of the 2000 columns.
3. **Diagonal** (top-left to bottom-right): Apply Kadane's to each of the $2 \times 2000 - 1 = 3999$ diagonals.
4. **Anti-diagonal** (top-right to bottom-left): Apply Kadane's to each of the 3999 anti-diagonals.

#### Kadane's Algorithm

For a sequence $a_1, a_2, \ldots, a_n$:

```
max_ending_here = 0
max_so_far = -infinity
for each a_i:
    max_ending_here = max(a_i, max_ending_here + a_i)
    max_so_far = max(max_so_far, max_ending_here)
return max_so_far
```

This runs in $O(n)$ time and $O(1)$ space.

## Editorial

After the lagged Fibonacci sequence has been generated, the problem becomes a repeated one-dimensional maximum subarray search. Every admissible contiguous subsequence lies on exactly one row, one column, one main diagonal, or one anti-diagonal, so it is enough to extract each line in those four families and run Kadane's algorithm on it.

The implementation follows that decomposition literally. It fills the $2000\times2000$ table row by row from the generator, then scans rows and columns directly. For the two diagonal directions, it enumerates all offsets, builds the corresponding line, and again applies Kadane's algorithm. Because Kadane's algorithm returns the best contiguous sum in linear time for a single sequence, repeating it across all four line families covers every possible direction while keeping the overall runtime quadratic in the grid size.

## Pseudocode

```text
Generate the four million sequence values and place them row by row into a 2000 by 2000 table.

Set the best answer to negative infinity.

For each row:
    run Kadane's algorithm and update the best answer.

For each column:
    collect the column entries, run Kadane's algorithm, and update the best answer.

For each top-left to bottom-right diagonal:
    collect its entries, run Kadane's algorithm, and update the best answer.

For each top-right to bottom-left diagonal:
    collect its entries, run Kadane's algorithm, and update the best answer.

Return the largest value seen in any of these scans.
```

## Complexity Analysis

- Generating the table: $O(N^2)$ where $N = 2000$.
- Kadane's over all directions: $O(N^2)$ per direction (each cell is visited once per direction).
- Total: $O(N^2) = O(4 \times 10^6)$.
