\section*{Overview}

Geometry notes will stay deliberately narrow: sturdy predicates first, then the contest routines that grow out of
them. The main goal is to avoid fragile floating-point reasoning when integer geometry is enough.

\section*{Scope}

The current coverage starts with the integer-geometry basics that most later routines depend on: orientation tests,
segment interaction, hull construction, and polygon classification. From there the natural next layer is calipers,
closest-pair style routines, and more global sweep-line reductions.
