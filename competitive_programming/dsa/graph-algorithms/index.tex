\section*{Overview}

Graph problems often become simpler once the hidden graph is made explicit. The notes here focus on that step:
identify the graph, identify the query pattern, then choose the graph routine that preserves the structure you care
about.

\section*{What I Care About in This Category}

The most reusable graph ideas are usually not the proofs in isolation but the reductions:

\begin{itemize}
  \item when a shortest-path routine is enough and when it is not,
  \item when a tree should be linearized,
  \item when the hard part is actually the path aggregation structure around the graph.
\end{itemize}
