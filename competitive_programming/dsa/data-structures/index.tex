\section*{Overview}

This branch collects the data structures I expect to reach for repeatedly in contest code: small range-query tools,
balanced trees, and dynamic structures that let an update stay local while the answer remains global.

\section*{How I Organize the Notes}

I try to separate the notes by the kind of invariant they maintain:

\begin{itemize}
  \item prefix-based structures such as Fenwick trees,
  \item interval-based structures such as segment trees,
  \item set / sequence BSTs such as treaps and splay trees,
  \item dynamic-forest structures such as link-cut trees.
\end{itemize}

That makes it easier to see which tool to try when the problem statement changes one assumption.
